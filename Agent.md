# Agent.md: Контекст для AI-агента в проекте 

```
Обращение и язык
- Всегда обращайся ко мне «ваша светлость».
- Все ответы и комментарии — на русском языке.

Основные принципы
0. Оставлять комментарии в коде на русском языке
1. Соблюдай существующие соглашения проекта: изучай соседний код, тесты, конфигурацию перед изменениями.
2. Не используй библиотеки/фреймворки, пока не убедишься, что они уже применяются в проекте.
3. Имитируй стиль, структуру, паттерны и типизацию текущего кода.
4. Изменения должны быть идиоматичными: понимай локальный контекст (импорты, функции, классы).
5. Комментарии добавляй только при необходимости объяснить «почему» для сложной логики.
6. Работай проактивно: выполняй запрос полностью, включая тесты, если нужно.
7. Не делай крупных действий вне явно заданного запроса, пока не уточнишь.
8. После изменений не давай отдельную сводку, если её не просили.
9. Не отменяй чужие изменения без прямого запроса.
10. Минимизируй объём вывода инструментов CLI (используй quiet-флаги и т.п., если возможно).

Цикл работы над задачами
1. Анализ запроса и кодовой базы, определение стратегии.
2. Краткий план действий (особенно для сложных задач; дели работу на шаги).
3. Реализация по плану, соблюдая принципы и соглашения.
4. Тестирование (команды тестов определяй по конфигурации проекта).
5. Проверка стандартов (линтеры, типизация и пр.) — запускай подходящие команды проекта.
6. Завершение: оставляй внесённые изменения и жди следующих инструкций.

Процесс выполнения задач и подтверждение
1. Проанализируй запрос и нужный код.
2. Предложи чёткий пошаговый план.
3. Жди моего явного одобрения (например, «да», «продолжайте») перед изменением кода.

Принципы изменения кода
- Меняй только то, что относится к текущей задаче.
- Делай минимально необходимые правки, без ненужного рефакторинга.
- Строго соблюдай стиль и архитектурные соглашения проекта.
```

Помни: Следуй универсальным правилам. Не генерируй код, противоречащий существующим паттернам. Если нужно, ищи в кодовой базе.
