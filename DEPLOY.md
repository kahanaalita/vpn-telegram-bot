# Инструкция по деплою на Render.com

## Шаг 1: Создание GitHub репозитория

### 1.1 Создать репозиторий на GitHub
1. Откройте https://github.com/new
2. Введите название репозитория: `vpn-telegram-bot` (или любое другое)
3. Выберите **Public** или **Private**
4. **НЕ** добавляйте README, .gitignore, license
5. Нажмите **Create repository**

### 1.2 Загрузить код на GitHub

Выполните команды в терминале (в папке `vpn-bot`):

```bash
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ваш-username/vpn-telegram-bot.git
git push -u origin main
```

**Замените** `ваш-username` на ваш GitHub username!

---

## Шаг 2: Регистрация на Render.com

1. Откройте https://render.com
2. Нажмите **Get Started** или **Sign Up**
3. Выберите **Sign up with GitHub**
4. Авторизуйтесь через GitHub
5. Разрешите Render доступ к вашим репозиториям

---

## Шаг 3: Создание Web Service на Render.com

### 3.1 Создать новый сервис
1. В панели Render нажмите **New +** (справа вверху)
2. Выберите **Web Service**

### 3.2 Подключить GitHub репозиторий
1. Найдите репозиторий `vpn-telegram-bot` в списке
2. Нажмите **Connect**

### 3.3 Настроить сервис

Заполните поля:

| Поле | Значение |
|------|----------|
| **Name** | `vpn-bot` (или любое имя) |
| **Region** | Выберите ближайший регион (например, Frankfurt) |
| **Branch** | `main` |
| **Runtime** | `Go` |
| **Build Command** | `go build -o vpn-bot` |
| **Start Command** | `./vpn-bot` |
| **Instance Type** | `Free` |

### 3.4 Добавить переменную окружения (ВАЖНО!)

1. Прокрутите вниз до раздела **Environment Variables**
2. Нажмите **Add Environment Variable**
3. Заполните:
   - **Key:** `TELEGRAM_BOT_TOKEN`
   - **Value:** `ваш_токен_от_BotFather` (например: `8714284927:AAGdTbVcamDsjIx_baC54qk1PSA2n-LVqjI`)
4. Нажмите **Add**

### 3.5 Создать сервис
1. Нажмите **Create Web Service** внизу страницы
2. Дождитесь деплоя (2-5 минут)

---

## Шаг 4: Проверка

### 4.1 Проверить статус деплоя
1. На странице сервиса смотрите раздел **Logs**
2. Должна появиться строка: `Authorized on account @ваш_бот`
3. Статус должен быть **Live** (зелёный)

### 4.2 Проверить бота в Telegram
1. Откройте бота в Telegram
2. Отправьте `/start`
3. Отправьте `/configs`
4. Должны прийти VLESS конфигурации

---

## Особенности бесплатного тарифа Render.com

✅ **Плюсы:**
- Полностью бесплатно
- Автоматический перезапуск при падении
- Автодеплой при push в GitHub
- 750 часов/месяц

⚠️ **Минусы:**
- Сервис засыпает после 15 минут неактивности
- Первый запрос после сна может быть медленным (30 секунд)
- Для Telegram ботов это не критично - бот проснётся при первом сообщении

## Обновление бота

Чтобы обновить код бота:

1. Внесите изменения в `main.go`
2. Закоммитьте и запушьте на GitHub:
```bash
git add .
git commit -m "Update bot"
git push
```
3. Render автоматически задеплоит новую версию

---

## Отзыв токена (ВАЖНО ДЛЯ БЕЗОПАСНОСТИ!)

**Ваш токен был опубликован в чате!** Обязательно отзовите его:

1. Откройте @BotFather в Telegram
2. Отправьте `/mybots`
3. Выберите вашего бота
4. Нажмите **API Token**
5. Нажмите **Revoke current token**
6. Получите новый токен
7. Обновите переменную `TELEGRAM_BOT_TOKEN` на Render.com:
   - Зайдите в настройки сервиса
   - Раздел **Environment**
   - Измените значение `TELEGRAM_BOT_TOKEN`
   - Нажмите **Save Changes**
   - Сервис автоматически перезапустится

## Troubleshooting

### Бот не отвечает
- Проверьте логи на Render.com
- Убедитесь что переменная `TELEGRAM_BOT_TOKEN` установлена правильно
- Проверьте что сервис в статусе **Live**

### Ошибка "Conflict: terminated by other getUpdates"
- Остановите все локальные экземпляры бота
- Перезапустите сервис на Render.com

### Бот засыпает
- Это нормально для бесплатного тарифа
- Бот проснётся при первом сообщении (может занять 30 секунд)
